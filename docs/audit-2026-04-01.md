# VoltGrid Jobs — Full Site Audit
**Date:** 2026-04-01
**Scope:** UI/UX, copy, and SEO across all major pages
**Method:** Source code review against product-marketing-context, page-cro, copywriting, seo-audit, and site-architecture skill frameworks
**Status:** Audit only — no fixes applied

---

## Summary

VoltGrid is well-structured for its stage. The core job board mechanics work, SEO infrastructure (sitemap, robots, structured data on key pages) exists, and the employer-side copy is genuinely strong. The biggest gaps are: a homepage H1 that misidentifies the audience, thin/templated SEO landing pages, missing meta descriptions on transactional pages, a global em dash violation in copy, and several UX friction points around email capture order.

---

## Homepage (`/`)

### Broken / Wrong

- **H1 misidentifies the audience.** "You Build AI. We'll Find You the Job." — electricians and HVAC techs don't build AI. They build the infrastructure AI runs on. This is a clever line that will resonate with the founder but may alienate the exact trades worker who lands on the page from a Google search for "data center electrician jobs." The sub-headline corrects this ("Join trades workers earning $45–$85/hr...") but the H1 is the headline, and it's the wrong hook.
- **Double email ask before any content.** `NewsletterCaptureWidget` fires, then immediately `JobAlertInlineForm` fires — both are above the company logo bar and category grid. A new visitor sees two email capture prompts before they've seen a single job. This is premature and kills trust. Pick one or move both below the fold.
- **"Download free →" label on Salary Guide CTA is misleading.** The link goes to `/salary-guide` which gates content behind an email form — it doesn't trigger a file download. Visitors who click expecting a PDF and get a form will feel tricked.
- **Section headings are `<p>` tags.** "Featured Jobs", "Latest Jobs", and "Browse by Trade" sections all use `<p>` with uppercase styling instead of `<h2>` elements. This is a semantic HTML and accessibility failure. Screen readers won't recognize these as section headers.
- **`onMouseEnter={undefined}` left on the "Hire Trades Workers" Link** (line ~133). Dead code, remove it.
- **"Jobs loading soon." fallback** (empty state when no recent jobs). This shouldn't appear if the DB has jobs, but it's a bad user experience when it does — implies the site isn't ready.

### Weak

- **Category tiles have no icons.** Six plain text boxes for trade categories. Compare to the rest of the design which uses icons liberally. No visual differentiation between trades.
- **"Hire Trades Workers →" secondary CTA** is a weak label. Employers landing on the homepage won't recognize this as directed at them. "Post a Job →" is clearer.
- **No social proof from job seekers.** There are trust stats (job counts, companies) but no testimonials, application volume data, or "X electricians found jobs here" type signal.
- **H1 and meta title mismatch.** Meta title is "VoltGrid Jobs — Data Center & AI Infrastructure Jobs for Electricians, HVAC Techs..." but the H1 is "You Build AI. We'll Find You the Job." Google may not understand what the page is about.
- **OG image not configured in layout metadata.** `/public/og-image.png` exists but there's no `openGraph.images` in the root layout metadata. Social shares will have no preview image.

### Missing

- No FAQ section or FAQPage schema on the homepage (used by Google for rich results)
- No link to `/resources`, `/interview-prep`, or `/salary-guide` in the main content area (only in footer)
- No `/companies` index page to browse all hiring companies
- Social links missing from footer (no LinkedIn, X/Twitter)

## Header (global)

### Broken / Wrong

- **Inline `onMouseEnter`/`onMouseLeave` event handlers** on nav links and the Post a Job button (lines 48-49, 65-66, 73-74). CLAUDE.md explicitly prohibits this: "Never add onMouseEnter, onMouseLeave, or any JS event handlers... use Tailwind CSS hover: classes instead." The header is a client component so it won't throw a runtime error, but it violates the stated coding rules.
- **Header is entirely client-rendered.** The full header (logo, nav, post-job CTA) is inside a `'use client'` component that waits for `supabase.auth.getUser()` before rendering user state. Nav links appear immediately, but the auth-dependent items (Sign In vs Dashboard avatar) will flash. The real cost is that the header is excluded from SSR HTML for SEO crawlers.

### Weak

- **No active/current-page indicator** on nav links. A user on `/resources` has no visual confirmation of where they are.
- **"Sign In" label** doesn't communicate value. "My Jobs" or "Dashboard" would tell returning users what they get.
- **No keyboard-accessible focus styles** visible in the source code for nav links.

### Missing

- No skip-to-main-content link for accessibility

## Jobs Listing Page (`/jobs`)

### Broken / Wrong

- **H1 defaults to "All Jobs"** when no filter is active. "All Jobs" is one of the worst possible SEO H1s — no keyword signal, no differentiation, Google has nothing to rank. Should be "Data Center Trades Jobs" or "Data Center Electrician, HVAC & Low Voltage Jobs."
- **H1 and meta title mismatch.** Meta title: "Browse Data Center & AI Infrastructure Jobs." H1 (unfiltered): "All Jobs." These should align.
- **Full jobs table scanned for filter dropdown.** The page makes 3 DB queries: a count, a paginated fetch, and then `select company_name, category from jobs where is_active = true` for ALL active jobs to build the company filter. At scale this will degrade. Should use a GROUP BY aggregate query or precomputed top-N.
- **`ilike` on description field** in the keyword search filter. Without a GIN index, this is a full-table scan. Flag as a future performance risk.

### Weak

- **No sort options.** Users can filter but can't sort by salary or post date. "Highest salary first" would be high-value for this audience.
- **No breadcrumb.** The page is an orphan in the navigation hierarchy (no visual path showing where you are).
- **No schema markup.** A `ItemList` or `JobPosting` aggregate schema would help Google understand the page.
- **Filter sidebar has no "clear all filters" shortcut** at the top (only individual clear actions).
- **Pagination creates potential duplicate content.** `/jobs?page=2` and `/jobs?category=electrical&page=1` have no canonical tags differentiating them from the base `/jobs` URL.

### Missing

- Canonical tags on paginated/filtered views
- `ItemList` schema
- Sort controls (salary, date)
- SavedSearch/alert prompt higher up (currently only at bottom after all results)

## Employers Page (`/employers`)

### Broken / Wrong

- **FAQ answer publicly admits scraping.** The FAQ "Do you just pull jobs from other sites?" answers: "Yes, for now — that is how we built the initial catalog." This is honest but damaging for employers evaluating whether their $149 gets them unique reach. Consider rewording or removing this FAQ until the ratio of direct posts to scraped jobs improves.
- **"Launch pricing — rates increase after April 30" urgency strip** — today is April 1, 2026. This strip has a hard expiry in 29 days. If not removed or updated post-April 30, it becomes a broken promise and a trust signal in the wrong direction.
- **`Image` imported but unused** in the component (line 2: `import Image from 'next/image'`). Dead import.
- **Em dash violations in copy.** Multiple instances violate VoltGrid Voice Rules ("No em dashes"):
  - "VoltGrid is niche — every candidate on the platform is a trades professional" (FAQ: "What's the difference from Indeed?")
  - "VoltGrid is a vertical board — the only candidates here are the ones you actually want." (objection handler)
  - These should use commas or colons.

### Weak

- **No employer testimonials or case studies.** The "Why pay for VoltGrid?" section uses hypothetical ROI math instead of real hiring stories. One "We posted 3 roles and filled them in 2 weeks" quote would outperform all four objection cards combined.
- **Pro Monthly plan has only 3 feature bullets** vs. 4 for Single Post. The premium plan looks under-featured.
- **No path for enterprise/staffing firm conversations.** Staffing firms placing 20-50 workers per month won't self-serve at $799/month without talking to someone first. No "contact us" or "enterprise pricing" option.
- **Pain points reference competitor brand names** (Indeed, LinkedIn, Staffing Agencies) without data to substantiate the comparison claims.

### Missing

- Employer testimonials / social proof
- "Request a demo" or "Contact us" CTA for enterprise
- Quantified results ("X applications per listing on average")

## Post a Job (`/post-job`)

### Broken / Wrong

- **Meta description is missing.** Only `title` is set in the metadata export. Google will auto-generate a description from page content, which for a form-heavy page will be garbage.

### Weak

- **Static fallback `?? 350` in TrustRail** — if the DB count is 0 (shouldn't happen, but during outages), the page shows "350+ specialist trades jobs listed" as a hardcoded minimum. Fine in practice but should be `count ?? 0` with a different fallback string if 0.
- **Page is entirely delegated to `PostJobPageClient`** — the server page renders almost no SSR HTML. The trust rail and form are rendered client-side. This means Google's crawler sees a near-empty page for `/post-job`.

### Missing

- Meta description
- OG tags (title and description)
- Any server-rendered content visible to crawlers

## Resources Page (`/resources`)

### Broken / Wrong

- **`JobAlertInlineForm variant="homepage"` placed at the top of a resource browsing page** — this is a full-width email capture widget dropped between the hero and the resource cards. Users came to browse career resources and instead hit an email gate immediately. This is friction in the wrong place and will hurt resource page engagement. Move it to the bottom, or use the compact `variant="jobs"` version.
- **`var(--surface)` CSS variable** used on resource cards and CTA block. This variable doesn't appear in the standard design token set (which uses `--bg-raised`, `--bg-subtle`). If `--surface` is not defined globally, these elements will inherit an undefined background.

### Weak

- **H1 "Data Center Trades Knowledge Hub"** — not keyword-optimized. "Resources" in the meta title doesn't match "Knowledge Hub" in the H1. Neither targets a search query. A better H1: "Data Center Electrician & HVAC Career Resources."
- **3 coming-soon cards at 50% opacity** occupy a third of the resource grid and make the page look sparse. Either remove them or replace with links to relevant jobs by trade.
- **Resource cards don't show estimated read time** — a common pattern for resource hubs that helps users decide what to click.

### Missing

- BreadcrumbList JSON-LD schema (breadcrumb HTML exists but no structured data)
- Article schema on individual resource cards
- Internal links to trade/location landing pages from resource descriptions

## Trade + Location Landing Pages (`/[tradeLocationSlug]`)
*e.g., `/electrical-jobs-in-northern-virginia`*

### Broken / Wrong

- **Intro text is a hardcoded template for ALL locations.** The paragraph "X is one of the most active data center markets in the US" is identical for every location. If the slug is `hvac-jobs-in-topeka-ks`, Topeka is still "one of the most active data center markets." This is thin content that Google will flag.
- **No BreadcrumbList JSON-LD schema.** Breadcrumb HTML renders (`Home > Jobs > [Trade] in [Location]`) but there's no `BreadcrumbList` structured data. Google Search won't show breadcrumbs in results.
- **No JobPosting or ItemList schema** for the job listings shown on the page. These pages are built specifically to rank for "data center electrician jobs in Phoenix" — structured data is table stakes for those queries.
- **`notFound()` called when `jobs.length === 0 && !locationName`** — legitimate, but means the `generateStaticParams` (top 60 combos with ≥2 jobs) could generate pages that 404 if jobs expire between build and visit.

### Weak

- **`tradeDef.blurb` is identical across all locations** for a given trade. "Journeymen, master electricians, and apprentices wiring data centers and AI infrastructure." is the same whether the page is for Dallas or Denver.
- **Hard cap of 50 jobs, no pagination.** If a trade/location combo has 60+ listings, 10+ are invisible.
- **Alert widget is below the fold when jobs exist.** Jobs list renders first, then the alert widget. For the majority of pages which will have few or no jobs, the alert path should be prominent.
- **Employer CTA section uses `<p>` instead of an `<h2>`.** "Hiring [trade]s in [location]?" deserves a proper heading.

### Missing

- Unique, location-specific content (salary ranges, major employers, local certification info)
- Pagination
- `ItemList` or `JobPosting` JSON-LD schema
- BreadcrumbList JSON-LD schema
- H2 subheadings structuring the page (the entire page has only an H1)

## Global / Cross-Cutting Issues

### SEO

1. **Sitemap includes `/jobs?category=X` query-param URLs.** Google doesn't reliably crawl query-parameter URLs from sitemaps. These should be removed from the sitemap; the canonical equivalents are `/trades/[slug]` pages.
2. **No OG image configured globally.** `og-image.png` exists in `/public` but is not set in the root `layout.tsx` metadata. Every social share will be imageless.
3. **Sitemap.ts makes 2 separate limit-5000 queries** for trade-location pages and hire-trade-location pages, fetching the same `category, location` data twice. Can be deduplicated.
4. **Company slug generation in sitemap.ts** iterates up to 5000 jobs in JS to count by company. Should use `GROUP BY company_name` in Postgres.

### Copy Violations (VoltGrid Voice Rules)

Em dashes found in production copy — all violate the "No em dashes" rule:
- `/employers`: "VoltGrid is niche — every candidate on the platform..."
- `/employers`: "VoltGrid is a vertical board — the only candidates here..."
- `/employers`: "On VoltGrid, that's $12,000–$20,000 per hire" (range dash, different from em dash — this is fine)

### Accessibility

1. No skip-to-main-content link anywhere on the site.
2. Multiple client components (`InterviewPrepGate`, `JobAlertInlineForm`, `AlertSignupWidget`) return `null` on initial render while checking localStorage/auth — causes cumulative layout shift (CLS) on every page load.
3. Header auth-dependent rendering means the nav skeleton and final state differ, contributing to CLS.

### Architecture / Navigation

1. **Footer is missing links to**: `/resources`, `/interview-prep`, `/companies`, `/break-into-data-center-work`. The footer's "Job Seekers" column only links to category filter pages.
2. **Duplicate pricing destination**: `/pricing` page exists AND `/employers#pricing` section exists. Footer links to `/pricing` under employers. Potential for these to drift out of sync.
3. **No `/companies` index** despite the route tree having `companies/[companySlug]/`. This is a dead-end subtree with no entry point from navigation.
4. **`/trades/[slug]` and `[tradeLocationSlug]` are parallel SEO hierarchies** for the same queries. The sitemap includes both. There's a risk of internal cannibalisation if they target the same keywords.
